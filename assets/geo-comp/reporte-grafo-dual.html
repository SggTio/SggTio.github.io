1. Introducción

Este reporte describe, paso a paso, la teoría, los algoritmos y la implementación del grafo dual de un polígono simple. El problema central es:

Problema

Dado un polígono simple P, triangularlo y construir su grafo dual: un grafo donde cada triángulo se convierte en un nodo, y dos nodos se conectan si sus triángulos comparten una diagonal interior. Los triángulos que tocan el borde del polígono se conectan a un nodo especial P∞ que representa la cara exterior (el “infinito”).

El pipeline completo que implementamos es:

  1. Lectura del polígono desde un archivo .obj
  2. Garantía de orientación CCW (antihoraria)
  3. Triangulación mediante partición y-monótona + sweep line
  4. Cálculo de baricentros de cada triángulo
  5. Construcción del grafo dual (mapa de aristas, clasificación borde/interior, P∞)
  6. Visualización por capas con VTK (PNGs, GIF, modo interactivo)

2. Polígono simple

Un polígono simple es una figura plana cerrada formada por una secuencia finita de segmentos de recta (aristas) que no se intersecan entre sí, excepto en sus vértices consecutivos. Formalmente: dados n puntos v0, v1, …, vn-1, el polígono es la unión de los segmentos vivi+1 mod n.

Nuestros archivos de entrada usan el formato Wavefront OBJ simplificado: líneas v x y z para vértices (z siempre 0) y una línea f i j k … con los índices base-1 de la cara. 

# Ejemplo: cuadrado CCW
v 0.0 0.0 0.0
v 4.0 0.0 0.0
v 4.0 4.0 0.0
v 0.0 4.0 0.0
f 1 2 3 4

3. Orientación y área con signo

La orientación de un polígono se define por el orden en que se recorren sus vértices. Existen dos convenciones:

  • CCW (Counter-Clockwise / Antihorario): el interior queda a la izquierda del recorrido. Produce área positiva.
  • CW (Clockwise / Horario): el interior queda a la derecha. Produce área negativa.

3.1 Fórmula del área con signo (Shoelace)

Calculamos el área con signo usando la fórmula del cordón de zapato (shoelace formula):

Fórmula de Shoelace

A = ½ ∑i=0n-1 (xi · yi+1 − xi+1 · yi)

donde los índices son módulo n. Si A > 0 ⇒ CCW. Si A < 0 ⇒ CW.

En nuestra implementación (Polygon.hxx):

TReal a = 0;
for( auto pIt = this->topology_begin(); pIt != this->topology_end(); ++pIt )
{
  auto qIt = std::next( pIt );
  if( qIt == this->topology_end() )
    qIt = this->topology_begin();

  auto p = this->m_Points[ *pIt ];
  auto q = this->m_Points[ *qIt ];

  a += p[0] * q[1];
  a -= q[0] * p[1];
}
return( a * TReal(0.5) );

3.2 Garantía CCW

La triangulación requiere orientación CCW. Si el área resulta negativa, invertimos el orden de los vértices: 

void guarantee_CCW()
{
  if( this->area() < TReal(0) )
    this->invert();  // std::reverse sobre los vértices de la topología
}

4. Triangulación del polígono

Triangular un polígono es descomponerlo en triángulos que no se solapan y que cubren toda su superficie. Todo polígono simple de n vértices produce exactamente n − 2 triángulos.

4.1 Estrategia: partición y-monótona + sweep line

Nuestra implementación usa un enfoque en dos fases:

Fase 1 — Partición Y-Monótona

Un polígono es y-monótono si cualquier línea horizontal lo corta en a lo sumo dos puntos. Usamos CGAL::y_monotone_partition_2 para descomponer el polígono original en piezas y-monótonas. Esto se realiza en O(n log n).

Fase 2 — Sweep Line (barrido vertical)

Cada pieza y-monótona se triangula con un algoritmo de barrido (sweep) que procesa los vértices de arriba hacia abajo, usando una pila para determinar qué diagonales insertar. Esto se realiza en O(n) por pieza.

4.2 El algoritmo de sweep (detalle)

Para una pieza y-monótona:

  1. Ordenar los vértices por coordenada y descendente (de arriba a abajo). En caso de empate, por x ascendente.
  2. Clasificar en cadenas: cada vértice pertenece a la cadena izquierda o derecha del polígono monótono (la función mark_left recorre la frontera desde el vértice más alto hasta el más bajo por un lado).
  3. Apilar los primeros dos vértices.
  4. Para cada vértice vj restante (excepto el último):
    • Si vj está en la misma cadena que el tope de la pila: desapilar vértices mientras formen diagonales válidas (test de orientación con CGAL::orientation) y emitir un triángulo por cada vértice desapilado.
    • Si está en la cadena opuesta: conectar vj con todos los vértices de la pila (emitir un triángulo por cada par consecutivo).
  5. Último vértice: conectar con todos los vértices restantes en la pila.
// Polygon_Functions.hxx — extracto del sweep
for( ; std::next(vIt) != sweep.end(); ++vIt )
{
  auto sv = s.top();
  if( std::get<2>(*vIt) == std::get<2>(sv) ) // Misma cadena
  {
    // desapilar mientras la diagonal sea válida
    while( valid && s.size() > 0 )
    {
      auto ori = CGAL::orientation( ... );
      if( diagonal_valida )

5. Baricentros

El baricentro (o centroide) de un triángulo con vértices A, B, C es el punto donde se intersecan sus tres medianas. Se calcula como el promedio aritmético de las coordenadas:

Fórmula del baricentro

G = ( (xA + xB + xC) / 3 , (yA + yB + yC) / 3 )

En dual_graph.cxx:

std::vector< TPoint > barycenters;
for( auto tIt = mesh.topology_begin(); tIt != mesh.topology_end(); ++tIt )
{
  const TPoint& pa = mesh.point( (*tIt)[0] );
  const TPoint& pb = mesh.point( (*tIt)[1] );
  const TPoint& pc = mesh.point( (*tIt)[2] );
  barycenters.push_back(
    TPoint(
      ( pa[0] + pb[0] + pc[0] ) / 3.0,
      ( pa[1] + pb[1] + pc[1] ) / 3.0
    )
  );
}

Cada baricentro se convierte en un nodo del grafo dual. Está siempre dentro de su triángulo, lo cual garantiza que los nodos del grafo dual estén siempre contenidos en el polígono original.

6. Construcción del grafo dual

El grafo dual transforma la topología de la triangulación en un grafo de adyacencia. La idea central:

  • Cada triángulo → un nodo (ubicado en su baricentro)
  • Cada arista compartida entre dos triángulos → una arista interna del dual
  • Cada arista de frontera (solo un triángulo la toca) → una arista externa hacia P∞

6.1 Algoritmo paso a paso (build_dual_graph)

La función build_dual_graph en DualGraph.hxx sigue estos pasos:

Paso 1 — Inicializar

Limpiar el objeto dual. Copiar los baricentros. Contar n triángulos.

Paso 2 — Construir el mapa de aristas (edge_map)

Para cada triángulo Tk con vértices (a, b, c), registrar sus tres aristas como pares ordenados (min, max) en un mapa: arista → [lista de triángulos].
Cada arista aparecerá en 1 o 2 triángulos. 

TEdgeMap edge_map;
for( auto tIt = mesh.topology_begin(); ... ; ++tIt, ++tri_idx )
{
  const TIndex a = (*tIt)[0], b = (*tIt)[1], c = (*tIt)[2];

  edge_map[ {min(a,b), max(a,b)} ].push_back( tri_idx );
  edge_map[ {min(b,c), max(b,c)} ].push_back( tri_idx );
  edge_map[ {min(a,c), max(a,c)} ].push_back( tri_idx );
}

Paso 3 — Clasificar aristas

Recorrer el edge_map. Si una arista tiene 2 triángulos → arista interna del dual (conecta los dos nodos). Si tiene 1 triángulo → arista externa (ese triángulo es de frontera, se conecta a P∞). 

for( const auto& entry : edge_map )
{
  const auto& tris = entry.second;
  if( tris.size() == 2 )  // Arista compartida → interna
  {
    dual.m_AdjacencyMatrix[i][j] = true;
    dual.m_InternalEdges.push_back( {i, j} );
  }
  else if( tris.size() == 1 )  // Arista frontera → externa
  {
    dual.m_IsBoundary[i] = true;
    dual.m_ExternalEdges.push_back( {i, p_inf_idx} );
  }
}

7. El nodo P∞ (punto en el infinito)

En la teoría de grafos planares, el grafo dual de una subdivisión planar incluye un nodo para la cara exterior (la región no acotada que rodea al polígono). Llamamos a este nodo P∞.

Todo triángulo que tiene al menos una arista sobre el borde del polígono (arista de frontera) es adyacente a la cara exterior. En el grafo dual, esto se refleja como una arista desde ese triángulo hacia P∞.

7.1 Posición de P∞

P∞ representa un punto “en el infinito”. Para la visualización, necesitamos ubicarlo fuera del polígono. Lo calculamos a partir de la caja envolvente (bounding box): 

// Bounding box del polígono
TReal cx = (xmin + xmax) / 2;
TReal cy = (ymin + ymax) / 2;
TReal dx = xmax - xmin;
TReal dy = ymax - ymin;

// Posicionar P∞ lejos, abajo a la derecha
dual.m_PointInfinity = TPoint( cx + 2*dx, cy - 2*dy );

La posición exacta no afecta la topología del grafo — solo sirve para que la visualización muestre las aristas externas de forma clara, sin interferir con las aristas internas.

Propiedad clave

El grafo dual de una triangulación de un polígono simple es siempre un árbol (no tiene ciclos). El árbol tiene exactamente n − 2 nodos internos (uno por triángulo) más el nodo P∞. Los nodos hoja del árbol (si los hubiera) corresponden a triángulos “orejas” del polígono.

8. Matriz de adyacencia

La estructura DualGraph almacena una matriz de adyacencia booleana de tamaño (n+1) × (n+1), donde n es el número de triángulos y la fila/columna n corresponde a P∞.

M[i][j] = true indica que el triángulo i y el triángulo j comparten una arista (o que i está conectado a P∞ si j = n).

Ejemplo: cuadrado (2 triángulos)

Un cuadrado se triangula en 2 triángulos: T0 y T1. Comparten una diagonal (arista interna). Ambos tienen aristas de frontera (aristas externas a P∞).

9. Implementación

9.1 Arquitectura de la librería pujCGAL

La implementación sigue un diseño header-only con templates de C++17. Cada clase se divide en .h (declaración) y .hxx (implementación), incluido automáticamente al final del .h.

9.2 Kernel de CGAL

Usamos Exact_predicates_inexact_constructions_kernel: predicados geométricos exactos (orientación, intersección) pero construcción de puntos con aritmética de punto flotante. Esto da robustez numérica sin sacrificar rendimiento.

9.4 Formato de salida del grafo dual

El archivo .obj del grafo dual usa v para los baricentros (P∞ como último vértice) y l para las aristas: 

# Dual graph: 4 nodes, 2 internal edges, 3 external edges
v 3.33 2.00 0     ← baricentro T0
v 2.00 0.67 0     ← baricentro T1
v 0.67 2.00 0     ← baricentro T2
v 10.0 -6.00 0    ← P∞ (último vértice)

l 1 2              ← arista interna T0—T1
l 2 3              ← arista interna T1—T2
l 1 4              ← arista externa T0—P∞
l 2 4
l 3 4

El ejecutable visualizer es una aplicación C++ que usa VTK (Visualization Toolkit) para renderizar las tres capas (polígono, triangulación, grafo dual) con colores consistentes.

10.2 Tres modos de operación

  • Modo por defecto (sin flags): genera 5 PNGs paso a paso en output/result-NNN/. Cada ejecución crea una carpeta nueva, sin pisar resultados anteriores.
  • --gif: genera los mismos 5 PNGs más un dual_graph.gif animado usando ffmpeg con la técnica de concat demuxer + palettegen para calidad óptima.
  • --interactive: abre una ventana VTK en vivo con controles: teclas 1/2/3 muestran/ocultan capas, s captura pantalla, q sale.

10.4 Herramienta de dibujo: polygon_drawer.py

Script Python/VTK interactivo para crear polígonos a mano y exportarlos como .obj. Usa una ventana con proyección ortográfica 2D, glyphs esféricos para los vértices, y soporte para deshacer, limpiar y guardar.
